# Architecture

Implement the following architecture.

Retrieval Results

↓

KnowledgeIntelligenceService

↓

KnowledgePipeline

↓

KnowledgeProcessor[]

↓

ContextPackage

KnowledgeIntelligenceService must only orchestrate the pipeline.

Business logic belongs inside processors.

---

# Core Abstractions

Implement the following abstractions.

## KnowledgeContext

Represents the mutable processing context shared across all processors.

Example fields

- query
- retrieved_chunks
- ranked_chunks
- citations
- relationships
- conflicts
- timeline
- metadata
- statistics
- context_package

Processors update this object.

---

## KnowledgeProcessor

Abstract processor.

Every processor must implement

process(context)

Each processor performs exactly ONE responsibility.

Processors must not depend on each other directly.

---

## KnowledgePipeline

Responsible for

- executing processors
- maintaining execution order
- passing KnowledgeContext
- returning final ContextPackage

The pipeline should allow processors to be:

- added
- removed
- reordered
- replaced

without modifying KnowledgeIntelligenceService.

---

# Default Pipeline

Implement the default pipeline.

1. AuthorityRankingProcessor
2. VersionResolutionProcessor
3. ConflictDetectionProcessor
4. RelationshipExpansionProcessor
5. DuplicateRemovalProcessor
6. CitationProcessor
7. TimelineProcessor
8. ContextBuilderProcessor

Each processor should only modify the KnowledgeContext.

# Authority Ranking

Implement

AuthorityRankingProcessor

Responsibilities

- assign authority scores
- rank enterprise documents

Default authority order

Law

↓

Government Decree

↓

Central Bank Circular

↓

Internal Regulation

↓

Internal SOP

↓

Guideline

↓

FAQ

Authority rules must be configurable.

---

# Version Resolution

Implement

VersionResolutionProcessor

Responsibilities

- detect document versions
- prefer active versions
- resolve effective dates
- preserve historical versions

Do not delete historical versions.

---

# Conflict Detection

Implement

ConflictDetectionProcessor

Detect

- contradictory regulations
- conflicting versions
- duplicated policies

Return structured conflict information.

Do not attempt automatic resolution.

---

# Relationship Expansion

Implement

RelationshipExpansionProcessor

Expand retrieval using

- parent documents
- child documents
- referenced documents
- appendices
- replaced regulations
- related documents

Expansion depth must be configurable.

Prevent infinite recursion.

---

# Duplicate Removal

Implement

DuplicateRemovalProcessor

Remove duplicate chunks using

- chunk id
- document id
- semantic similarity

Keep only the highest ranked chunk.

---

# Citation Builder

Implement

CitationProcessor

Every citation should include

- document title
- document id
- page
- section
- chunk
- authority
- version
- effective date

Citation generation must be deterministic.

---

# Timeline Builder

Implement

TimelineProcessor

Generate chronological document history.

Example

Circular 01/2020

↓

Circular 18/2022

↓

Circular 35/2025

Timeline generation should be optional.

---

# Context Builder

Implement

ContextBuilderProcessor

Merge

- ranked chunks
- metadata
- citations
- relationships
- timeline

Produce

ContextPackage

No Prompt Engineering.

No LLM formatting.
