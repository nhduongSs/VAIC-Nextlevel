# Objective

Implement a production-ready retrieval engine capable of searching enterprise knowledge using both lexical and semantic search.

The output of this wave is a ranked list of relevant document chunks.

# Vector Retriever

Implement

VectorRetriever

Responsibilities

- Generate query embedding
- Search pgvector
- Return nearest chunks

Use the existing EmbeddingProvider from Wave 2.3.

Support:

- Cosine similarity
- Configurable Top K
- Similarity threshold

---

# Hybrid Retriever

Implement

HybridRetriever

Responsibilities

- Execute BM25 retrieval
- Execute Vector retrieval
- Merge results
- Normalize scores
- Remove duplicates
- Produce unified ranking

Support configurable weights.

Example

Hybrid Score

=

α × Vector Score

+

β × BM25 Score

Weights must be configurable.

---

# Score Normalization

Implement score normalization.

Supported strategies:

- Min-Max Normalization

Design for future extension.

# API

Implement REST endpoints.

POST /search

GET /search/health

POST /search/preview

Example request

{
  "query": "...",
  "top_k": 10,
  "filters": {
      "document_type": "...",
      "authority_level": "...",
      "language": "vi"
  }
}

Example response

{
  "query":"...",
  "results":[
      {
          "chunk_id": "...",
          "document_id":"...",
          "score":0.93,
          "retrieval_method":"hybrid",
          "metadata":{...},
          "content":"..."
      }
  ]
}

Do NOT return embeddings.
